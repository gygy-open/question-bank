# 版本与选型

Question Bank 提供两种部署形态，**功能完全一致**，区别在于**运行载体**与**适用规模**。

## 版本模型

```
Question Bank
├── 桌面版 (Windows 托盘应用)
│   ├── 个人使用      —— 绑定 127.0.0.1，仅本机访问，单人
│   └── 局域网共享    —— 托盘开启，绑定 0.0.0.0 + 防火墙放行，同网段多人访问
└── 服务器版 (Docker Compose)
    —— MySQL + ChromaDB + Backend + Worker + Frontend，面向团队 / 生产
```

## 对比一览

| 维度 | 桌面版 · 个人使用 | 桌面版 · 局域网共享 | 服务器版 |
|------|------------------|--------------------|----------|
| 运行载体 | Windows 桌面 App | Windows 桌面 App | 服务器 / Docker |
| 安装方式 | 安装包一键安装 | 安装包一键安装 | Docker Compose |
| 数据库 | SQLite（内置） | SQLite（内置） | MySQL 8.0 |
| 访问范围 | 仅本机 | 同一局域网 | 局域网 / 内网 / 公网（自配） |
| 多用户 | 单人为主 | 支持，多人共用一份数据 | 支持，面向团队 |
| 数据目录 | `%APPDATA%/QuestionBank` | `%APPDATA%/QuestionBank` | 服务器卷 / MySQL |
| 升级方式 | 覆盖安装 | 覆盖安装 | 拉取新镜像重建 |
| 维护成本 | 极低 | 低 | 中（需运维 Docker/MySQL） |
| 适合人群 | 个人教师、备课 | 教研组、机房、小团队 | 学校 / 机构 / 生产部署 |

## 该用哪个？

- **只在自己电脑上用** → 桌面版 · [个人使用](/desktop/personal)
- **几个人在同一间办公室 / 机房共用一份题库** → 桌面版 · [局域网共享](/desktop/lan-sharing)
- **团队规模较大、需要长期在线、要接入内网 / 公网、需要更强的并发与备份** → [服务器版](/server/docker)

::: tip 数据可迁移
桌面版与服务器版共用同一套数据模型。随着团队变大，可从桌面版迁移到服务器版（导出题目 / 迁移数据库），无需重新录题。具体迁移步骤见 [数据库与迁移](/server/database)。
:::

## 关于「局域网共享」与「服务器版」的区别

两者都能多人使用，但定位不同：

- **局域网共享**是桌面版的一个开关： 把本机的桌面 App 临时开放给同网段的同事访问，依赖你这台电脑开机； **默认关闭**，适合轻量协作。
- **服务器版**部署在一台长期在线的服务器上，使用 MySQL，具备更好的并发、稳定性与备份能力，适合作为团队的正式题库服务。
