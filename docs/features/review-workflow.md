# 审核工作流

题目通过一套状态流转进行质量把关，并完整记录审核日志。

## 状态流转

```
草稿(draft) → 待审(pending) → 发布(published) → 归档(archived)
```

| 状态 | 标识 | 含义 |
|------|------|------|
| 草稿 | `draft` | 已创建，尚未提交审核 |
| 待审 | `pending` | 已提交，等待审核 |
| 发布 | `published` | 审核通过，正式可用 |
| 归档 | `archived` | 不再使用，留档 |

## 审核次数与阈值

- 每个题目有 `review_count`（审核次数），每次审核操作递增。
- 每个学科可设置 `required_review_count`（默认 1），作为发布前的最低审核门槛。

在 [学科与标签](/features/subjects-and-tags) 中可配置学科的审核阈值。

## 审核日志

- 每次审核动作记录到活动日志（`action = review`），包含审核人、时间与状态变化。
- 结合 [操作审计](/features/activity-logs) 可追溯完整历史。

## 权限

- 超级管理员可审核全部内容。
- 普通用户对自己的内容进行相应操作。

关于角色与权限详情，见 [用户与权限](/admin/users)。
